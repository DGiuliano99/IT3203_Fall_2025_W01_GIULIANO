    Key Concepts

    • HTTP is a client-server protocol in which a client, a web browser or other web processing code, sends a request to a server, or collection of servers.
    • In the late 1980's, Tim Berners-Lee developed HTTP based on HyperText, a method of displaying documents in which links, called “hyperlinks”, can be accessed by the client
    • HTTP protocol has continued to develop over time with major releases HTTP/2 and HTTP/3, released in 2015 and 2022 respectively, continuously adding additional features that improve the speed, efficiency, and accuracy of data transmission.
    • While the primary components of HTTP are the client and server, the reality is that there are many devices between the client and the server or collection of servers hosting a resource. The devices between the client and the server are called proxies.
