    REST API's and HTTP Requests

    HTTP requires consistent formatting to allow for cross compatibility, and for that reason both requests and responses have specific formatting requirements. HTTP requests since version one have included the HTTP version the client is capable of reading. The request also includes the URL for the server resource. This will allow the client to later lookup the IP of the resource.

    HTTP requests include request headers, which are key pairs of information that contain additional data about the request. For "POST" requests, there also may be a request body that is transmitted that contains the data payload to be uploaded to the server. HTTP responses are also subject to similar formatting requirements, however, have slightly different components.

    HTTP requests are most used with Representation State Transfer (REST) Application Programming Interfaces (API). These are custom built scripts that are highly configurable to meet the needs of the system. Because these scripts are customizable, they can be crafted to accomplish a wide variety of business functions.

    This allows systems and applications to communicate with the server using POST and GET requests primarily to access and modify data on the server. These APIs can be used in conjunction with HyperText pages to create integrations. A major downside to REST APIs is that each individual application requires its own unique coding to be implemented.

    This contrasts with WebDAV which used a set of common protocols that allowed for increased interoperability between applications. This can lead to additional development time for new integrations.

    HTTP Request Methods

    Method Description
    GET Contacts the server asking for information.
    POST Contacts the server with information for the server to process.
    UPDATE Variant of POST; Contacts the server with database updates.
    DELETE Variant of POST; Contacts the server to delete database data.
